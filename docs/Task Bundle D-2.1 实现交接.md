## 1. 本轮交接目标
在不扩写到运行策略管理、审计摘要与模型连通性测试的前提下，先完成 **Task Bundle D-2 的后台配置子集**，让“后台模型配置是唯一业务配置源”真正形成可见闭环。

本轮只做三件事：
1. 提供后台模型配置读写入口
2. 在管理端补齐最小模型配置页面
3. 让前台在“未配置”时明确提示并阻止新开局，而不是静默失败

一句话目标：
> 先把“后台模型配置真的能配、前台没配时真的会说清楚”这件事做实。

## 2. 对应任务范围
### 对应 Tasks
- T51【US5】实现后台模型配置与未配置状态管理

### 当前实际切片
本轮交付不是整个 Task Bundle D / E，而是其第一段：**D-2.1 / 后台模型配置与未配置状态切片**。

### 明确包含
- 提供管理员读取当前模型配置的 API
- 提供管理员更新 `decision` / `narrative` 模型配置的 API
- 配置项至少覆盖：模型名称、Base URL、API Key 掩码提示、思考强度、启用状态
- 管理端前端从骨架推进到可填写、可保存的最小模型配置页面
- 登录态与 `/api/auth/me` 返回当前模型运行状态摘要
- 前台新开局前检查是否存在可用模型配置；若无，则明确提示并阻止新开局
- 为上述链路补齐必要接口、共享类型、前后端展示与测试
- 更新必要的运行 / 验证说明与状态文档

### 明确不包含
- 运行策略管理页
- 审计摘要展示页
- 模型连通性测试
- 外部真实模型调用接入
- 用户管理之外的后台大重构
- D-1 范围回流扩写
- 将当前切片扩成完整后台平台

## 3. 交付完成定义
只有同时满足以下条件，才算 D-2.1 完成：

1. **后台模型配置成立**
   - 管理员可读取当前 `decision` / `narrative` 配置状态
   - 管理员可保存模型名称、Base URL、API Key、思考强度与启用状态
   - API Key 不完整回显，只以掩码提示展示

2. **管理端页面成立**
   - 后台前端不再停留在纯骨架说明
   - 管理员可在页面完成最小模型配置操作

3. **未配置状态闭环成立**
   - 登录态或 `/api/auth/me` 能返回当前模型运行状态摘要
   - 当前无可用模型配置时，前台新开局会被明确阻止
   - 提示语义清楚，不是静默失败

4. **验证闭环成立**
   - 至少有与上述三块对应的自动化验证或集成验证
   - D-1 与既有主链路无明显回归

5. **文档与状态回写成立**
   - `sdd-status.md` 已同步
   - 必要时更新 `review.md` 或补充 D-2.1 的验收记录入口

## 4. 建议实现顺序
1. 先补共享类型与服务端模型配置读写 API
2. 再补登录态 / `/api/auth/me` 的模型运行状态摘要
3. 再补前台新开局前的未配置阻断与提示
4. 最后补管理端模型配置页面与交互
5. 补齐测试并做主链路回归
