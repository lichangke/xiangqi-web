## 1. 本轮交接目标
在不扩写到真实模型接入、真实 API Key 持久化与连通性测试的前提下，先完成 **Task Bundle D-2 的第二段**，让后台从“仅能配模型入口”继续推进到“可调系统策略、可看关键审计摘要”。

本轮只做三件事：
1. 提供运行策略读写入口
2. 在管理端补齐运行策略管理页面 / 区块
3. 提供轻量审计摘要查询与展示

一句话目标：
> 先把“后台不只是能配模型，也能调系统策略并看到关键操作痕迹”这件事做实。

## 2. 对应任务范围
### 对应 Tasks
- T52【US5】实现运行策略与轻量审计摘要

### 当前实际切片
本轮交付不是整个 Task Bundle D / E，也不是“真实模型接入最小闭环”，而是其第二段：**D-2.2 / 运行策略与审计摘要切片**。

### 明确包含
- 提供管理员读取当前 `RuntimePolicy` 的 API
- 提供管理员更新关键运行策略的 API
- 运行策略至少覆盖：AI 对局并发上限、单账号同时进行中的对局数上限、注册策略开关与系统默认值
- 审计日志记录模型配置修改、策略修改等关键后台操作
- 提供轻量审计摘要查询 API
- 管理端前端补齐运行策略管理与轻量审计摘要展示
- 为上述链路补齐必要接口、共享类型、前后端展示与测试
- 更新必要的运行 / 验证说明与状态文档

### 明确不包含
- 真实 API Key 安全持久化
- 真实外部模型调用接入
- 模型连通性测试
- 更细粒度的 `decision` / `narrative` 运行期消费链路
- 用户管理之外的后台大重构
- D-2.1 范围回流扩写
- 将当前切片扩成完整后台平台

## 3. 交付完成定义
只有同时满足以下条件，才算 D-2.2 完成：

1. **运行策略成立**
   - 管理员可读取当前运行策略
   - 管理员可保存关键运行策略
   - 策略更新后可被后端读取并实际生效于主链路约束（至少不回退现有 `maxOngoingGamesPerUser`）

2. **管理端页面成立**
   - 后台前端可查看并修改运行策略
   - 后台可查看关键审计摘要，而不只是写入不展示

3. **审计摘要闭环成立**
   - 关键后台操作会写入 `AuditLog`
   - 管理端可查询并展示轻量审计摘要

4. **验证闭环成立**
   - 至少有与上述三块对应的自动化验证或集成验证
   - D-2.1 与既有主链路无明显回归

5. **文档与状态回写成立**
   - `sdd-status.md` 已同步
   - 必要时更新 `review.md` 或补充 D-2.2 的验收记录入口
