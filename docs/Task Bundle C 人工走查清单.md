# Task Bundle C 人工走查清单

## 文档定位
- 文档类型：最小人工验收清单
- 适用范围：Task Bundle C（演绎展示与移动端体验）
- 当前状态：有效
- 最后更新时间：2026-03-27

## 走查目标
用最小成本确认 Bundle C 的产品层关键体验没有明显跑偏。

## 建议环境
- 电脑浏览器一轮
- 手机竖屏一轮
- 手机横屏一轮（或浏览器设备模拟横屏）

## 账号建议
- 演示账号：demo / demo123

## 一、登录与开局
### C-01 登录
- 操作：登录 demo 账号
- 期望：进入用户端页面，无报错

### C-02 新建对局
- 操作：分别选择一个难度并创建新局
- 期望：棋盘正常显示；页面提示为已创建对局；讨论区可准备展示回合内容

## 二、主链路与回合演绎
### C-03 合法落子
- 操作：点击己方棋子，再点击目标点完成一次合法落子
- 期望：
  - 用户落子成功
  - AI 能返回应对
  - 页面出现新的“第 N 回合”演绎卡
  - 最新回合应按分段逐步展开，而不是一次性全挤出来

### C-04 回合骨架检查
- 操作：观察最新回合演绎内容
- 期望：可识别出统一四段结构：
  - 简评
  - 发言
  - 共识
  - 落子

## 三、特殊事件模板
### C-05 非法落子提示
- 操作：尝试一次明显非法的落子
- 期望：
  - 棋局状态不被污染
  - 出现“非法落子”类提示
  - 提示区分于普通系统错误

### C-06 悔棋提示
- 操作：点击悔棋
- 期望：
  - 最近一个完整回合被撤回
  - 事件区出现“悔棋事件”提示
  - 页面提示当前轮到你重新落子

### C-07 认输提示
- 操作：点击认输并确认
- 期望：
  - 对局结束
  - 事件区出现“认输事件”或终局相关提示
  - 棋盘与状态区进入结束态

### C-08 将军 / 终局反馈
- 操作：若对局过程中出现将军或直接打到终局，观察提示
- 期望：
  - 将军时出现独立提醒
  - 终局时出现终局反馈

## 四、主题切换
### C-09 三主题切换
- 操作：依次切换 classic / ink / midnight
- 期望：
  - 页面即时切换
  - 不需要刷新
  - 棋盘、讨论区、事件区层级不乱

### C-10 主题边界确认
- 操作：退出登录或刷新页面后观察
- 期望：
  - 当前不要求保留主题
  - 若刷新后不保留，不算问题

## 五、移动端体验
### C-11 竖屏可用
- 操作：手机竖屏或设备模拟竖屏打开页面
- 期望：
  - 棋盘为视觉中心
  - 讨论区默认折叠或不会长期压住棋盘
  - 主要操作仍可完成

### C-12 横屏更佳
- 操作：手机横屏或设备模拟横屏打开页面
- 期望：
  - 布局比竖屏更舒展
  - 讨论区 / 状态区可更顺手地查看
  - 不影响棋盘操作

### C-13 讨论区折叠 / 展开
- 操作：在移动端来回折叠、展开讨论区
- 期望：
  - 不会挡住主要棋盘操作
  - 展开后能看到演绎、状态、设置内容

## 六、收口判定
若以下条件都满足，可视为人工走查通过：
- 登录与新局正常
- 合法落子后演绎能出现且分段展开
- 非法落子 / 悔棋 / 认输至少三类模板工作正常
- 三主题切换即时生效
- 手机竖屏可用、横屏更佳
- 讨论区默认折叠策略未明显影响主链路

## 七、若发现问题，建议记录格式
发现问题时，建议至少记录：
- 场景（电脑 / 手机竖屏 / 手机横屏）
- 操作步骤
- 实际结果
- 期望结果
- 是否阻断主链路
