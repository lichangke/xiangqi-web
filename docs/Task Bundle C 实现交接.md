## 3. 本轮必须完成

### 3.1 独立标准决策层与结构化解释落地
在当前最小合法对局闭环之上，落地独立标准决策层（规则 + 启发式评分），保证机器落子与结构化解释成体系输出。

#### 必需字段
- `chosenMove`
- `userMoveTag`
- `aiMoveTag`
- `situationShift`
- `turnArc`
- `storyThreadSummary`

#### 可选字段
- `highlightReason`
- `riskLevel`
- `pressureSide`

要求：
- 决策层负责“为什么这么下”
- 不直接生成最终给用户看的剧情文案
- 若决策层失败，不得破坏已有合法落子闭环，必须退回稳定兜底路径

### 3.2 普通回合与特殊事件统一进入同一时间线
必须让前端只保留一套时间线容器，同时承载：
- 普通回合 `turn`
- 特殊事件 `event`

本轮重点确保以下事件真正进入演绎时间线，而不是只停留在提示层：
- `illegal_move`
- `undo`

同时校验或补齐以下事件在统一 schema 下的表现一致性：
- `resign`
- `check`
- `finish`

要求：
- 非法步与悔棋必须形成独立时间线项
- 特殊事件不能只是改个标题、实质仍像普通回合
- 事件事实必须先由系统给出，AI 不拥有事实判定权

### 3.3 演绎输入输出契约落地
实现过程中，演绎层必须遵守已确认的输入输出契约：
- 系统先构造结构化 `NarrativeRequestEnvelope`
- 演绎层只消费结构化解释、主题上下文、角色上下文与剧情线程摘要
- 前端消费统一时间线项，而不是直接消费原始生成结果
- 生成结果失败、超时、schema 非法时，必须 fallback，但时间线不能断

### 3.4 `storyThreadSummary` 以系统规则化方式维护
本轮必须把 `storyThreadSummary` 当作系统维护的轻量状态对象，而不是一段随意长文。

最低要求：
- 只保留 1 份当前线程状态摘要
- 由最近 2～4 回合滚动更新形成
- 默认控制在 80～150 字，上限 200 字
- 包含以下 5 个槽位：
  - `currentPhase`
  - `mainConflict`
  - `pressureSide`
  - `recentFocus`
  - `carryForward`

要求：
- V1 先采用系统规则化更新
- 不引入新的轻量 summarizer AI 链路
- 更新失败时允许退回上一版摘要，不得阻断主链路

### 3.5 演绎质量提升与成本降级策略落地
必须让演绎质量相较上一轮明显减轻“呆板感”，但不能靠失控文案或扩大系统边界来解决。

要求：
- 普通回合与高光事件使用同一供应商 / 同一模型族
- 差异主要通过字数、语气、角色表现强度拉开
- 特殊事件与高光时刻优先投入预算
- 普通回合响应不能明显拖慢主链路
- 记录“失败原因 + 回合类型 + 事件类型”的组合日志
- fallback 保留少量世界观气质，但必须克制、贴事实、不断时间线

### 3.6 保持上一轮已通过部分不回退
本轮不是重做整个 Bundle C，而是在现有技术完成度基础上修正体验闭环。

因此要求：
- 合法落子主链路不能退化
- 主题即时切换不能被破坏
- 手机端竖屏可用 / 横屏更佳的已有基础不能被新链路打坏
- 不因演绎升级而把问题外溢到 Bundle D 范围
