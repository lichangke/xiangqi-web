## 1. 本轮交接目标
在不扩写到后台模型配置、运行策略与审计摘要的前提下，先完成 **Task Bundle D 的用户侧闭环子集**，让账号体系真正具备持续使用价值。

本轮只做三件事：
1. 主题偏好持久化与恢复
2. 最近对局摘要
3. 同账号跨端登录但同一时刻仅允许 1 局进行中的对局限制

一句话目标：
> 先把“账号真的有持续使用价值”这件事做实。

---

## 2. 对应任务范围
### 对应 Tasks
- T41【US4】实现主题系统、偏好保存与最近对局摘要
- T42【US4】实现同账号跨端登录与单局限制

### 当前实际切片
本轮交付不是整个 Task Bundle D，而是其第一段：**D-1 / 用户数据闭环切片**。

### 明确包含
- 将当前已存在的三主题即时切换能力接入持久化存储
- 用户重新登录后恢复主题偏好
- 若项目中已有其他基础展示偏好字段，允许在不明显扩范围的前提下顺手纳入同一偏好持久化链路；若当前并不存在，则不为了“凑完整”新增太多偏好项
- 记录最近对局摘要，并在合适入口展示最小可用摘要信息
- 实现同账号跨端登录允许，但在尝试新开对局时校验“同一时刻最多 1 局进行中”
- 为上述链路补齐必要接口、存储结构、前后端展示与测试
- 更新必要的运行 / 验证说明与状态文档

### 明确不包含
- 后台模型配置
- 未配置状态闭环
- 运行策略管理
- 轻量审计摘要
- 外部大模型接入
- Bundle C 的额外文案打磨
- 对主题系统做超出持久化所需的大改造
- 复杂对局列表页或完整历史战绩系统

---

## 3. 交付完成定义
只有同时满足以下条件，才算 D-1 完成：

1. **主题偏好持久化成立**
   - 用户修改主题后可持久保存
   - 用户重新登录后能恢复主题偏好

2. **最近对局摘要成立**
   - 系统能记录最近对局的最小摘要信息
   - 用户可在现有合理入口看到最近对局摘要

3. **跨端单局限制成立**
   - 同账号允许多端登录
   - 若已有进行中的对局，再尝试新开局时会被明确阻止
   - 提示语义清楚，不是静默失败

4. **验证闭环成立**
   - 至少有与上述三块对应的自动化验证或集成验证
   - 主链路回归未被破坏

5. **文档与状态回写成立**
   - `sdd-status.md` 已同步
   - 必要时更新 `review.md` 或补充 D-1 的验收记录入口

---

## 4. 建议实现顺序
1. 先补用户偏好与最近对局摘要所需的数据结构 / 持久化字段
2. 再打通主题偏好保存与恢复
3. 再补最近对局摘要读写与前端展示
4. 最后实现“同账号仅允许 1 局进行中”的创建对局限制与提示
5. 补齐测试并做主链路回归
