## 1. 本轮交接目标
在 D-2.7 已按“当前边界内通过 / 外部访问样本作为后续增强项”口径正式收口的前提下，开启新一轮 **外部访问侧样本 / 扩展 burn-in**。

本轮不再回答以下旧问题：
1. `codex.hiyo.top` 应该走哪种协议；
2. provider 是否真的能命中；
3. `responses` 最终对象为空是否代表没有正文；
4. D-2.6 / D-2.7 的 contract / implement / review / 稳定快照是否已经形成。

这些问题都已收口。本轮要回答的新问题是：
1. 当前客户端 / 管理端 / 服务端在**外部网络 / 外部设备**视角下，是否能持续访问与完成最小主链路；
2. 当前 decision / narrative 在更多样本、更多回合、更多事件类型下，是否仍稳定保持目标运行口径；
3. 当前 narrative 是否在扩展 burn-in 中出现 fallback、结构退化、语义漂移或明显体验下降；
4. 当前是否已经暴露出值得从“稳定性增强证据轮”进一步升级为新问题切片的信号。

一句话目标：
> 在不回退 D-2.7 已收口结论的前提下，把问题主轴切换到 **外部访问侧证据补强 + 扩展 burn-in**，并形成新的可执行入口。
