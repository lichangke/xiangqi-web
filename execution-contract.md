## 1. 执行目标
本轮执行协议服务于 V1 实现的第二批核心工作，目标是完成 Task Bundle B：核心对局主链路与 AI 回合闭环，在已完成的基础框架之上跑通“新建对局 → 用户合法落子 → AI 合法应对 → 局面更新 → 悔棋 / 认输基础支持”的最小可运行闭环。

## 2. 本轮执行范围
### 明确包含
- 补全 GameSession 对局推进主链路与用户落子 API
- 在服务端完成合法落子校验、局面更新与回合状态落盘
- 基于当前 RuleAdapter + `elephantops` 实现 AI 合法候选步选择闭环
- 建立最小前端棋盘页，支持新建对局、点击选子、点击目标点落子与局面刷新
- 实现悔棋 / 认输基础能力，并保持与已确认规则约束一致
- 补齐本 bundle 对应的集成测试与必要人工冒烟验证

### 明确不包含
- 回合演绎文本与逐段出现展示
- 特殊事件模板包装（将军 / 终局 / 悔棋 / 认输的表现层模板）
- 手机端布局精修与移动端专项体验优化
- 三主题切换、偏好保存、最近对局摘要
- 后台模型配置、运行策略、审计摘要、未配置状态闭环
- 长将 / 长捉 / 循环和棋等规则附录专项补完

### 对应任务范围
- 任务组：任务组 B
- Task Bundle：Task Bundle B
- 用户故事：US1（核心对局主链路）、US3（悔棋 / 认输基础闭环部分）
- 对应任务：T12、T13、T14、T31

## 5. 汇报与返回规则
### 默认应返回
- 完成 Task Bundle B 后
- 出现关键阻塞时
- 出现需求冲突 / 方案歧义时
- 对局状态机或前后端协议与 spec 冲突时
- 当前 bundle 只能部分完成时

### 默认不返回
- 局部代码修改
- 普通调试过程
- 小尝试失败
- 单条命令细节

### 立即返回条件
出现以下情况时必须立即返回：
- 关键前提不成立
- 需要改动 Plan 级方案
- 需要扩大已确认范围
- 任务定义不足以继续执行
- 需要用户拍板
- 当前代码现状与文档严重不一致
- 核心验证失败且无法快速收敛

### 返回内容
- 当前卡点 / 当前完成情况
- 已尝试动作
- 可选方案
- 推荐方案
- 已完成项 / 未完成项
- 修改文件列表
- 执行命令
- 验证结果
- 风险 / 阻塞

## 7. 测试与验证要求
### 主链路验证
- 用户可新建一局并进入对局页面
- 用户可完成一次合法落子
- AI 可返回合法应对并推动局面更新
- 悔棋 / 认输基础能力可用

### 任务级验证
- T12 → 新建对局、读取当前对局、落子处理接口可用
- T13 → AI 最终落子来自合法候选步集合
- T14 → 前端最小棋盘页可完成“选子 → 落子 → 刷新局面”主链路
- T31 → 悔棋规则、次数限制与认输基础闭环可验证

### 边界 / 异常验证
- 非法落子不会污染棋局状态
- AI 不出现非法步
- 悔棋不能连续使用且不超过每局上限
- 同步检查前后端对局状态字段是否保持一致

### 最低完成标准
- 无验证结果，不算完成
- 无法执行验证时，必须说明原因与当前影响范围
