## 1. 执行目标
本轮执行协议服务于 V1 实现的 **Task Bundle D-2.1（后台模型配置与未配置状态）**。

D-1 已完成 review 收口并形成稳定快照。当前不回流扩写 D-1，也不把整个 D-2 / E / F 一次性打满，而是先交付其第一段高性价比切片：
1. 后台模型配置管理入口
2. 模型配置持久化与启用开关
3. 前台“未配置”状态提示与阻断新开局闭环

本轮目标不是运行策略完整管理，不是审计摘要页面，不是模型连通性测试，而是：
> 先把“后台模型配置是唯一业务配置源，且未配置时前台不会静默失败”这件事做实。

## 2. 本轮执行范围
### 明确包含
- 提供管理员读取当前模型配置的 API
- 提供管理员更新 `decision` / `narrative` 模型配置的 API
- 配置项至少覆盖：模型名称、Base URL、API Key 掩码提示、思考强度、启用状态
- 管理端前端补齐最小模型配置页面，不再停留在纯骨架
- 登录态与 `/api/auth/me` 返回当前模型运行状态摘要
- 前台新开局前检查是否存在可用模型配置；若无，则明确提示并阻止新开局
- 为上述链路补齐必要测试、状态回写与最小 UI 提示

### 明确不包含
- 运行策略完整管理页
- 审计摘要展示页
- 模型连通性测试
- 外部真实模型调用接入
- 用户管理之外的后台大重构
- D-1 范围回流扩写
- 将当前切片扩成完整后台平台

### 对应任务范围
- 任务组：任务组 E（仅 D-2.1 切片）
- Task Bundle：Task Bundle D-2.1
- 当前轮次定位：Task Bundle D-2.1 / Implement handoff
- 对应任务：T51（后台模型配置与未配置状态子集）

### 主链路验证
- 管理员可读取当前 `decision` / `narrative` 配置状态
- 管理员可保存并启用模型配置
- API Key 不完整回显，只返回掩码提示
- 登录态与 `/api/auth/me` 能返回模型运行状态摘要
- 当前无可用模型配置时，前台新开局会被明确阻止且提示清楚

### 回归验证
至少执行并回报：
- 与模型配置、登录态、前台新开局限制直接相关的测试
- `npm test`
- `npm run build`

- 无验证结果，不算完成
- 无法执行验证时，必须说明原因、影响范围与剩余风险
