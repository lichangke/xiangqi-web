## 2. 本轮执行范围
### 明确包含
- 建立单仓全栈项目骨架
- 设计数据库初稿与基础实体结构
- 完成用户名密码认证基础能力
- 接入规则适配层 baseline（优先评估 xiangqi.js）
- 补齐典型规则测试样例
- 为后续 GameSession 和 AI 主链路提供统一规则接口

- 后台模型配置页面
- 最近对局摘要与偏好恢复
- 审计摘要、运行策略配置、未配置状态完整闭环

### 对应任务范围
- 任务组：任务组 A
- Task Bundle：Task Bundle A
- 用户故事：US1（部分基础能力）、US2（规则基线部分）、US4 / US5 / US6（底座部分）
- FR：FR-001、FR-002、FR-010、FR-011、FR-012、FR-013，以及为 FR-023 ~ FR-031 提供基础承载

## 5. 汇报与返回规则
### 默认应返回
- 完成 Task Bundle A 后
- 出现关键阻塞时
- 出现需求冲突 / 方案歧义时
- 规则 baseline 与 spec 冲突时
- 当前 bundle 只能部分完成时

### 默认不返回
- 局部代码修改
- 普通调试过程
- 小尝试失败
- 单条命令细节

### 立即返回条件
出现以下情况时必须立即返回：
- 关键前提不成立
- 需要改动 Plan 级方案
- 需要扩大已确认范围
- 任务定义不足以继续执行
- 需要用户拍板
- 当前代码现状与文档严重不一致
- 核心验证失败且无法快速收敛

### 返回内容
- 当前卡点 / 当前完成情况
- 已尝试动作
- 可选方案
- 推荐方案
- 已完成项 / 未完成项
- 修改文件列表
- 执行命令
- 验证结果
- 风险 / 阻塞

## 6. Coding Agent 权限边界
### 允许
- 按已确认 tasks 与当前 task bundle 执行
- 小范围必要调整
- 极小型必要任务补充（需记录）
- 合理局部重构
- 运行必要验证

### 不允许
- 擅自修改已确认 Spec
- 擅自扩大范围
- 引入大重构
- 跳过验证直接宣称完成
- 把历史会话中的未确认内容当成当前要求

## 7. 测试与验证要求
### 主链路验证
- 项目基础框架可启动
- 用户名密码登录链路可打通
- RuleAdapter 可生成合法走法并判断非法步

### 任务级验证
- T01 → 前后端基础骨架与启动命令可用
- T02 → schema 能初始化且覆盖关键实体
- T11 → 典型规则样例通过
- T12 → 新建对局 / 读取对局接口可用

### 边界 / 异常验证
- 非法步典型场景验证
- 规则适配层与业务层边界验证
- 用户名密码认证失败路径验证

### 最低完成标准
- 无验证结果，不算完成
- 无法执行验证时，必须说明原因与当前影响范围
