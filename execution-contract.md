## 1. 执行目标
本轮执行协议服务于 V1 实现的第三批核心工作，目标是完成 Task Bundle C：在 Task Bundle B 已跑通最小合法对局闭环的基础上，升级独立标准 AI 决策层，并落地 AI 驱动的结构化演绎体验，让产品同时具备“机器更会下、演绎更成体系、事件能统一进时间线、手机可用、主题可切”的最小成品感。

---

## 2. 本轮执行范围
### 明确包含
- 重构 / 接入独立标准 AI 决策层，用于替代当前仅满足最小闭环的策略层能力，并直接改善人机对战体验
- 决策 AI 在生成机器落子的同时，稳定输出结构化解释，至少覆盖：`chosenMove / userMoveTag / aiMoveTag / situationShift / turnArc / storyThreadSummary`
- 实现普通回合演绎展示结构，形成统一的“第 N 回合”展示骨架
- 实现逐段出现效果，并控制节奏、容器与样式，使普通回合展示具备稳定可读性
- 补齐特殊事件统一 schema、事件语义标签与展示区分，至少覆盖：非法落子、悔棋、认输、将军、终局反馈
- 完成手机端主链路可用性优化：竖屏可用、横屏更佳、棋盘为视觉中心、讨论区默认折叠、提示层不长期遮挡棋盘
- 保持“点击选子 + 点击目标点”在手机端顺手可用，不引入拖拽交互
- 完成三主题前端视觉实现与设置入口切换，并支持当前页面即时生效

### 明确不包含
- 无上限棋力追求、复杂训练体系、搜索型引擎化深度改造
- 后台可配置的演绎模型或长文本复杂生成链路
- 三主题偏好持久化、重新登录恢复、最近对局摘要
- 后台模型配置、运行策略、审计摘要、未配置状态闭环
- 长将 / 长捉 / 循环和棋等规则附录专项补完
- 原生 App 级体验、复杂手势、拖拽落子、深度动画优化

### 对应任务范围
- 任务组：任务组 C
- Task Bundle：Task Bundle C
- 用户故事：US1（决策层增强部分）、US2（提示表现层部分）、US3（演绎与特殊事件模板）、US4（主题前端切换部分）
- 对应任务：T13（独立决策层增强轮次）、T21（展示消费与必要小补充）、T32、T33、T41（仅三主题前端切换部分）

## 5. 汇报与返回规则
### 默认应返回
- 完成 Task Bundle C 后
- 出现关键阻塞时
- 出现需求冲突 / 方案歧义时
- 演绎结构、事件模板或手机端交互与 spec 冲突时
- 当前 bundle 只能部分完成时

### 默认不返回
- 局部样式修改
- 普通调试过程
- 小尝试失败
- 单条命令细节

### 立即返回条件
出现以下情况时必须立即返回：
- 关键前提不成立
- 需要改动 Plan 级方案
- 需要扩大已确认范围
- 任务定义不足以继续执行
- 需要用户拍板
- 当前代码现状与文档严重不一致
- 核心验证失败且无法快速收敛

### 返回内容
- 当前卡点 / 当前完成情况
- 已尝试动作
- 可选方案
- 推荐方案
- 已完成项 / 未完成项
- 修改文件列表
- 执行命令
- 验证结果
- 风险 / 阻塞

## 7. 测试与验证要求
### 主链路验证
- 独立 AI 决策层能稳定生成机器落子，并产出约定的结构化解释字段
- 普通回合具备统一演绎展示骨架与逐段出现效果
- 非法落子、悔棋、认输、将军、终局能通过统一 schema 进入同一时间线
- 手机端竖屏可用、横屏更佳，讨论区默认折叠且不长期遮挡棋盘
- 三主题可在设置入口中切换，并在当前页面即时生效

### 任务级验证
- T13（增强轮次）→ 决策层可独立产出 `chosenMove / userMoveTag / aiMoveTag / situationShift / turnArc / storyThreadSummary`
- T21（展示层部分）→ 非法落子提示能稳定展示且不打断主链路
- T32 → 悔棋 / 认输 / 将军 / 终局事件可通过统一 schema 进入演绎时间线
- T33 → 普通回合逐段出现与统一结构模板可用
- T41（前端切换部分）→ 三主题视觉切换即时生效，当前不要求持久化

### 边界 / 异常验证
- 提示层不长期遮挡棋盘关键区域
- 手机端“点击选子 → 点击目标点”操作链路顺手可用
- 主题切换不会破坏棋盘、讨论区与事件展示层级
- 不因演绎或主题层引入对局主链路回归

### 最低完成标准
- 无验证结果，不算完成
- 无法执行验证时，必须说明原因与当前影响范围
