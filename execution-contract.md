## 1. 执行目标
本轮执行协议服务于 V1 实现的 **Task Bundle D-1（用户数据闭环）**。

上一轮 Task Bundle C 已完成 review 收口。当前不继续回头打磨 Bundle C，也不直接把整个 Bundle D 一次性打满，而是先交付其第一段高性价比切片：
1. 主题偏好持久化与恢复
2. 最近对局摘要
3. 同账号跨端登录但同一时刻仅允许 1 局进行中的对局限制

本轮目标不是后台配置闭环，不是运行策略，不是审计摘要，而是：
> 先把“账号真的有持续使用价值”这件事做实。

## 2. 本轮执行范围
### 明确包含
- 将当前三主题即时切换能力接入持久化存储
- 让用户重新登录后恢复主题偏好
- 在不明显扩范围的前提下，补齐当前最小用户偏好持久化链路
- 记录最近对局摘要，并在合适入口提供最小展示
- 实现同账号多端登录允许，但创建新对局时校验“最多 1 局进行中”
- 为上述链路补齐必要接口、持久化结构、前端消费与测试
- 做必要验证、回归验证与状态回写

### 明确不包含
- 后台模型配置
- 未配置状态管理
- 运行策略
- 审计摘要
- 外部大模型接入
- Bundle C 的额外文案打磨
- 复杂偏好中心
- 完整战绩系统 / 历史对局页
- 与当前切片无关的大重构

### 对应任务范围
- 任务组：任务组 D（仅 D-1 切片）
- Task Bundle：Task Bundle D-1
- 当前轮次定位：Task Bundle D-1 / Implement handoff
- 对应任务：T41、T42

## 5. 汇报与返回规则
### 默认应返回
- 完成 Task Bundle D-1 后
- 出现关键阻塞时
- 发现需要调整 Plan / Tasks / Contract 时
- 出现范围漂移风险时
- 当前切片只能部分完成时

### 默认不返回
- 普通调试过程
- 单条命令细节
- 局部样式修改
- 小范围失败尝试

### 立即返回条件
出现以下情况必须立即返回：
- 关键前提不成立
- 需要扩大已确认范围
- 需要改动 Plan 级方案
- 当前 task 定义不足以继续执行
- 需要用户拍板
- 当前代码现状与权威文档严重不一致
- 核心验证失败且无法快速收敛

### 返回内容
- 当前完成情况 / 当前卡点
- 已尝试动作
- 修改文件列表
- 执行命令
- 验证结果
- 未完成项 / 风险 / 阻塞
- 推荐下一步

## 6. Coding Agent 权限边界
### 允许
- 按当前已确认文档执行 Task Bundle D-1
- 做小范围必要调整
- 做合理局部重构
- 运行必要验证
- 修复实现中直接暴露的小问题（前提是不扩大范围）

### 不允许
- 擅自修改已确认 Spec
- 擅自进入 D-2（后台配置 / 审计 / 运行策略）
- 回头扩大 Bundle C 范围
- 跳过验证直接宣称完成
- 用历史会话里的未确认内容覆盖当前落盘文档

## 7. 测试与验证要求
### 主链路验证
- 主题切换后可持久保存
- 重新登录后可恢复主题偏好
- 最近对局摘要可生成并展示
- 多端登录本身不被错误阻止
- 已存在进行中对局时，再次新开局会被明确阻止
- Bundle C 已通过主链路无明显回归

### 任务级验证
- 偏好持久化链路具备最小自动化验证
- 最近对局摘要读写 / 展示具备最小自动化验证
- 单账号单进行中对局限制具备最小自动化验证
- 错误提示不是静默失败

### 回归验证
至少执行并回报：
- 与用户偏好 / 对局创建限制直接相关的测试
- `npm test`
- `npm run build`

### 人工验收建议
若条件允许，应最少验证：
1. 登录 → 切换主题 → 退出 / 重新登录 → 检查恢复
2. 开一局进行中对局 → 在另一端或等效场景尝试新开局 → 检查限制提示

### 最低完成标准
- 无验证结果，不算完成
- 无法执行验证时，必须说明原因、影响范围与剩余风险
